# AI Transparency

Purpose: disclose the AI tools, models, and automated components used in this project to ensure transparency, reproducibility, and responsible use.

## AI tools / platforms used
- OpenRouter (Arcee-AI Trinity Large Preview) — primary conversational LLM and translation assistant
- Groq (Whisper large-v3) — speech-to-text inference for voice input
- gTTS (Google Text-to-Speech) — text-to-speech for audio output
- HuggingFace (SEA-LION 7B Instruct) — optional regional-language refinement when `HF_TOKEN` is provided
- Development-time LLM assistance used for drafting and refactoring (internal use only)

## AI-generated components
- Chat responses and translations in the `ChatPanel` are generated by the Arcee-AI model via OpenRouter.
- Map greeting text and cultural-info phrasing may be refined by LLM outputs.
- Automatic phonetic/romanization prompts are produced by backend logic to improve TTS for script-incompatible languages.

## Custom engineering by the team
- Backend integration and routing: `backend/main.py` implements request orchestration, SSE streaming for chat, STT → LLM → TTS voice flows, and language fallback rules.
- Frontend UI and streaming client: `frontend/src/*` contains streaming client logic, audio recording pipeline, and UI components (ChatPanel, TalkPanel, DictionaryPanel, QuickPhrasePanel, MapPanel, QuizPanel).
- Language data curation: `backend/sea_languages_knowledge.json` and `backend/dictionary/*.json` were curated/edited by the team for regional accuracy.
- Fallback heuristics and TTS routing: language→TTS mapping (`LANG_CODE_MAP`, `WHISPER_LANG_MAP`) and romanization rules were authored by the team.

## Models / APIs integrated
- OpenRouter API — model: `arcee-ai/trinity-large-preview` (chat & translation)
- Groq API — model: `whisper-large-v3` (speech-to-text)
- HuggingFace Inference — model: `aisingapore/sea-lion-7b-instruct` (optional refinement)
- gTTS — Google text-to-speech library for audio generation

## Data handling and privacy
- Voice recordings and text inputs are processed in-session and not persisted by default. See `backend/main.py` for handling logic.
- No user accounts or long-term transcript storage is implemented. If persistent storage is added, it will be documented and opt-in.

## Limitations and responsible use
- LLM outputs may contain inaccuracies or culturally sensitive phrasing. Team review and curated knowledge are used to reduce harmful outputs but do not guarantee correctness.
- The system relies on third-party models and services (OpenRouter, Groq, HuggingFace, Google TTS). Their terms and privacy policies apply.

## Reproducibility
- To reproduce results locally, set required environment variables in `backend/.env` (see `.env.example`) and run the backend and frontend development servers.
- Example env vars: `OPENROUTER_API_KEY`, `GROQ_API_KEY`, `HF_TOKEN` (optional), `CORS_ORIGINS`, `VITE_API_BASE`.
